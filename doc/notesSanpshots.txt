procedure pour backup:
1. modifier elastic.yaml pour affecter un chemin pour les backup
path.repo: ['/usr/share/elasticsearch/backup']
docker-compose.yml a été modifié pour disposer d'un nouveau dossier (à la racine) backup et pour faire pointer ce dossier backup dessus
2. créer l'entrée pour snapshots (peut être fait avec Kibana mais pas OpenDashboards) :
a. avec curl : curl -X PUT "localhost:9200/_snapshot/backup?pretty" -H 'Content-Type: application/json' -d' {"type": "fs", "settings": {"location": "backup","compress": true}}'
b. vérifier que tout est bon
curl --user -X GET "localhost:9200/_snapshot/_all?pretty"

3. créer un snapshot
curl --user log:pass -X PUT "localhost:9200/_snapshot/backup/snapshot_001?wait_for_completion=true&pretty"

HACK
4. 	a. décompresser snap.tar.gz dans le dossier backup et écraser le contenu
	b. renommer les snap*.dat meta*.dat pour écraser les homonymes correspondant au snap du point 3 par ceux issus de la décompression
5. restaurer (j'ai fait avec Kibana donc j'ai pas le curl correspondant).
